## 2. 任务列表

### T01 · 逻辑与 / 逻辑或短路（30 分，入门）

**特性描述**：为表达式系统补齐 `&&` 与 `||`，并严格遵守短路求值。

**当前可运行的对照片段**：

```cangjie
let a = true
let b = false
if (a) {
    println(1)
}
if (b) {
    println(2)
}
return 0
```

**建议扩展后的目标写法（说明用途，不纳入当前自动执行）**：

```text
if (a && !b) { println(1) }
```

**期望效果**：支持逻辑组合表达式，并保证右侧表达式仅在必要时求值。

**实现方案提示**：
- 在 parser 中新增低于比较、高于赋值的优先级层级。
- 代码生成建议直接复用 `TEST + JMP` 模板实现短路，而不是退化成数值运算。

### T02 · 一元逻辑非 `!`（20 分，入门）

**特性描述**：新增布尔取反操作符。

**当前可运行的对照片段**：

```cangjie
let value = true
println(value)
return 0
```

**建议扩展后的目标写法（说明用途，不纳入当前自动执行）**：

```text
println(!value)
```

**期望效果**：支持布尔值求反，并在 parser 阶段对非 Bool 使用给出明确错误。

**实现方案提示**：
- 可将 `!expr` 归一化为新的 `UnaryOp.NOT`。
- Lua 5.5 已有 `NOT` 指令，若保持布尔结果可直接复用。

### T03 · `else if` 语法糖（25 分，入门）

**特性描述**：支持链式条件分支，避免深层嵌套块。

**当前可运行的对照片段**：

```cangjie
let flag = true
if (flag) {
    println(1)
} else {
    println(2)
}
return 0
```

**建议扩展后的目标写法（说明用途，不纳入当前自动执行）**：

```text
if (a) {
    println(1)
} else if (b) {
    println(2)
} else {
    println(3)
}
```

**期望效果**：解析器可连续处理 `else if`，代码生成仍输出扁平跳转链。

**实现方案提示**：
- 最简单做法是在 parser 中把 `else if` 递归转成嵌套 `IfStmt`。
- 文档中的字节码模板仍可复用现有 `if / else` 实现。

### T04 · `break` 语句（40 分，中等）

**特性描述**：允许提前终止最近一层循环。

**当前可运行的对照片段**：

```cangjie
var i = 0
while (i < 3) {
    println(i)
    i = i + 1
}
return 0
```

**建议扩展后的目标写法（说明用途，不纳入当前自动执行）**：

```text
while (i < 10) {
    if (i == 5) {
        break
    }
    i = i + 1
}
```

**期望效果**：循环体内可生成跳出当前循环的前向跳转，并在循环结束时统一回填。

**实现方案提示**：
- 为 `while` 维护 `break` 回填列表即可，不必一开始就实现多层标签。
- 若在非循环上下文看到 `break`，应直接报 parser error。

### T05 · `continue` 语句（45 分，中等）

**特性描述**：跳过当前迭代剩余语句，进入下一轮。

**当前可运行的对照片段**：

```cangjie
var i = 0
while (i < 3) {
    i = i + 1
    println(i)
}
return 0
```

**建议扩展后的目标写法（说明用途，不纳入当前自动执行）**：

```text
while (i < 5) {
    i = i + 1
    if (i == 2) {
        continue
    }
    println(i)
}
```

**期望效果**：支持回跳到循环条件计算点，而不是简单跳到循环体开头。

**实现方案提示**：
- 为每个循环保存“继续目标 PC”，让 `continue` 直接跳到条件重算位置。
- 注意 `continue` 前的栈临时寄存器要恢复。

### T13 · 闭包捕获局部变量（120 分，挑战）

**当前可运行的对照片段**：

**建议扩展后的目标写法（说明用途，不纳入当前自动执行）**：

**期望效果**：内部函数能够捕获上层局部变量，而不仅仅是共享 `_ENV`。

**实现方案提示**：
- 需要扩展 upvalue 描述，不再把所有子函数固定写成 `_ENV`。
- 建议先支持只读捕获，再考虑写回。

### T15 · 枚举与 `match`（130 分，挑战）
